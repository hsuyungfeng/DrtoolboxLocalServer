## Critical Anti-Patterns (do NOT repeat these)
- **[ANTI-PATTERN] Running llama-server without explicit GPU layer limit** → `--gpu-layers 50` causes CUDA crash. Always use `--gpu-layers 20` for this GPU/model combo. Verify with `--verbose 1` to check memory breakdown.
- **[ANTI-PATTERN] Using max_tokens < 100 with Qwen thinking model** → Thinking tokens consume output budget. For classification tasks, use keyword-based approach or set max_tokens ≥ 100.
- **[ANTI-PATTERN] Starting llama-server with bash tool timeout** → Tool kills background processes on timeout. Use dedicated startup script with PID tracking (see /tmp/start_llama.sh pattern).

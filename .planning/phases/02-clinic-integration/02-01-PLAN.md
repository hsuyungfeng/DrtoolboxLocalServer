  <name>Task 2: Query Queue + Retry Mechanism per D-01</name>
  <files>src/db/query_queue.py</files>
  <action>
Create `src/db/query_queue.py` implementing query queue with exponential backoff retry.

**Requirements:**
- Class `QueryTask`: Represents single query request with metadata
  - Attributes: id (uuid), query (str), params (tuple), created_at, retry_count, max_retries (default 3)
  - Attributes: last_error, last_error_time
- Class `QueryQueue`: Queue-based executor with built-in retry
  - Max queue size: 100 tasks (configurable, blocks on overflow)
  - Worker threads: 1-2 background workers (configurable, default 1)
  - Retry strategy: Exponential backoff (1s, 2s, 4s) + jitter
  - Per D-01: When HIS query fails (HISQueryTimeoutError, HISConnectionError), automatically retry up to 3 times before returning error
- Exported singleton: `get_query_queue()` returns thread-safe queue instance

**Signature:**
```python
@dataclass
class QueryTask:
    query: str
    params: tuple = ()
    max_retries: int = 3
    id: str = field(default_factory=lambda: str(uuid.uuid4()))
    created_at: datetime = field(default_factory=datetime.now)
    retry_count: int = 0
    last_error: Optional[Exception] = None
    last_error_time: Optional[datetime] = None

class QueryQueue:
    def __init__(self, num_workers: int = 1, max_queue_size: int = 100, 
                 his_connection: HISConnection = None):
        """Initialize query queue."""
    
    def submit(self, task: QueryTask) -> str:
        """Submit query task, return task.id."""
    
    def get_result(self, task_id: str, timeout: float = 10.0) -> List[Dict[str, Any]]:
        """Block until task completes or timeout, return result."""
    
    def status(self, task_id: str) -> Dict[str, Any]:
        """Get task status: pending, running, completed, failed."""

def get_query_queue() -> QueryQueue:
    """Get or create singleton QueryQueue."""
```

**Behavior (per D-01):**
1. Task enters queue
2. Worker picks up task
3. Worker calls `his_connection.execute(task.query, task.params)`
4. If HISQueryTimeoutError or HISConnectionError:
   - Increment retry_count
   - If retry_count < max_retries: re-queue with backoff delay (2^retry_count seconds + random jitter)
   - Else: mark task FAILED, return error to caller
5. If success: mark task COMPLETED, store result

**Logging:**
- Log queue depth every 10 submissions (info level)
- Log retry attempts (warn level): "Retrying query {task_id}, attempt {retry_count}"
- Log failures (error level): "Query {task_id} failed after {max_retries} retries: {error}"

**Testing hooks:**
- Accept mock HISConnection
- Ability to simulate HIS failures for retry testing
- Ability to inspect queue state for testing

**Schema Extension (clinic.db.sql):**
```sql
CREATE TABLE IF NOT EXISTS query_cache (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    query_hash TEXT UNIQUE NOT NULL,  -- SHA256(query + json.dumps(params))
    query_text TEXT NOT NULL,
    query_params TEXT,  -- JSON-encoded tuple
    result_json TEXT NOT NULL,  -- JSON-encoded result
    ttl_seconds INTEGER DEFAULT 3600,  -- Per D-02: 1 hour default
    cached_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP DEFAULT (datetime('now', '+1 hour')),
    hit_count INTEGER DEFAULT 0,
    is_valid BOOLEAN DEFAULT 1,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX IF NOT EXISTS idx_query_cache_hash
    ON query_cache(query_hash);

CREATE INDEX IF NOT EXISTS idx_query_cache_expires
    ON query_cache(expires_at);
```

**QueryCache class (src/db/query_cache.py):**
- `get(query, params) -> Optional[List[Dict]]`: Return cached result if valid (not expired), else None
- `set(query, params, result, ttl_seconds=3600)`: Store result in cache with TTL
- `invalidate(query=None, params=None)`: Clear cache entry or all entries
- `is_expired(query_hash) -> bool`: Check if cache entry exceeded TTL
- `auto_cleanup()`: Remove expired entries (run on startup or periodically)

**Signature:**
```python
class QueryCache:
    def __init__(self, db_connection):
        """Initialize cache with clinic.db connection."""
    
    def get(self, query: str, params: tuple = ()) -> Optional[List[Dict[str, Any]]]:
        """Get cached result if valid, else None."""
    
    def set(self, query: str, params: tuple, result: List[Dict], ttl_seconds: int = 3600):
        """Store result in cache with TTL."""
    
    def invalidate(self, query: str = None, params: tuple = None):
        """Invalidate cache entry or all entries."""
    
    def is_expired(self, query_hash: str) -> bool:
        """Check if entry exceeded TTL."""
    
    def auto_cleanup(self):
        """Remove expired entries."""
```

**Integration with QueryQueue:**
- QueryQueue will call `cache.get(task.query, task.params)` BEFORE submitting to HIS
- If cache hit: return cached result immediately (no queue submission)
- If cache miss: submit to HIS, then call `cache.set()` on success
- If HIS fails after cache miss: do NOT cache error (allow retry)

Per D-02: 1-hour TTL default for all queries. No per-query override needed for Phase 2 — all common queries use 1h.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Patient → LINE/API | Untrusted user input (patient queries, parameters) |
| External HIS → Internal DB | Trusted clinic IT database (read-only), but network may be unstable |
| API Response → Patient | Sensitive HIS data (patient names, appointments, medications) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | HIS connection | mitigate | Read-only mode enforced at connection layer (SQLite PRAGMA or MySQL grants); no INSERT/UPDATE/DELETE allowed |
| T-02-02 | Tampering | Query cache | mitigate | Cache table has read-only mode in clinic.db; no external writes permitted |
| T-02-03 | Information Disclosure | HIS patient data | mitigate | Patient data only returned via authenticated API endpoints (Phase 3); phase 2 is internal only. Query whitelist prevents arbitrary table access. |
| T-02-04 | Denial of Service | Query queue overflow | mitigate | Max queue size 100; overflow blocks caller until space available. Timeout on slow queries (5s) prevents hanging. Rate limiting in Phase 3. |
| T-02-05 | Denial of Service | HIS connection pool exhaustion | mitigate | Connection pool max 10; slow queries timeout after 5s releasing connection. Monitor queue depth in health check. |

**Security Posture Summary:**
- Read-only database access enforced at connection level (strongest defense)
- Query whitelist prevents SQL injection
- Connection timeouts prevent resource exhaustion
- Queue depth monitoring enables operational alerts
- No PII exposed in Phase 2 (internal HIS integration only; external API in Phase 3)

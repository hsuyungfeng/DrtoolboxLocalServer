## Key Patterns Applied

### DBContext Pattern
```python
class DBContext:
    """Context manager for guaranteed database connection cleanup"""
    def __enter__(self):
        self.conn = get_db_connection()
        return self.conn

    def __exit__(self, exc_type, exc_val, exc_tb):
        if self.conn:
            self.conn.close()
        return False
```

Applied to 13 endpoints across 3 files. Guarantees connection cleanup via `__exit__()` in all paths (success and exception).

### Input Validation Pattern
- **Type Checking:** `isinstance(patient_id, int) and patient_id > 0`
- **Format Validation:** `strptime(date_str, '%Y-%m-%d')`
- **Business Logic:** Future-date enforcement, enum validation
- **Length Limits:** `len(message_text) <= 1000`

### Query Optimization
Replaced:
```python
# N+1 pattern (bad - O(n) queries)
for patient in patients:
    cursor.execute('SELECT COUNT(*) FROM conversations WHERE patient_id = ?', ...)
```

With:
```python
# Aggregate pattern (good - O(1) query)
cursor.execute('''
    SELECT patient_id, COUNT(*) as msg_count
    FROM patient_conversations
    GROUP BY patient_id
''')
```

## Decision Rationale

1. **DBContext over manual cleanup:** Context manager pattern guarantees cleanup in all code paths (success, exception, early return). Eliminates the most common source of resource leaks.

2. **N+1 optimization:** Single GROUP BY aggregate reduces database round-trips from O(n) to O(1), critical for clinics with large patient counts (50+ patients).

3. **CR-01 deferral:** Authentication bypass requires full architectural redesign (hardcoded IDs → server-side sessions). Properly scoped for Phase 6 per code review assessment.

## Anti-Patterns to Avoid

- **Don't use:** Manual `conn.close()` without try/finally or context manager
- **Don't use:** Per-record queries in loops (N+1 pattern)
- **Don't use:** Unvalidated input from HTTP requests
- **Don't use:** Arbitrary field updates without whitelisting

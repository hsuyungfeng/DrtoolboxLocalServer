## Key Architectural Patterns

### DBContext Pattern (Resource Leak Prevention)
```python
class DBContext:
    def __enter__(self):
        self.conn = get_db_connection()
        return self.conn
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        if self.conn:
            self.conn.close()
        return False
```
Applied to 13 endpoints. Guarantees cleanup in all code paths (success, exception, early return).

### Input Validation (Defense in Depth)
- Type checking: `isinstance(patient_id, int) and patient_id > 0`
- Format validation: `strptime(date_str, '%Y-%m-%d')`
- Business logic: Future-date enforcement, enum validation
- Length limits: `len(message_text) <= 1000`

### Query Optimization (O(n) → O(1))
Replaced per-patient query loop with single GROUP BY aggregate in get_patient_stats().

### Field Whitelisting (SQL Injection Prevention)
Restricted updateable fields to {'appointment_date', 'status'} only.
